# promo

Drafts and ledgers for promoting balchat on open-source ecosystems.
None of this content is shipped with the app — it lives here so changes
are versioned and so anyone helping with outreach can copy from a
canonical place.

## What's where

```
promo/
  README.md                       you're here
  izzyondroid-request.md          paste-ready GitLab issue body for IzzyOnDroid
  awesome-tauri-pr.md             entry + PR description for tauri-apps/awesome-tauri
  awesome-rust-pr.md              same for rust-unofficial/awesome-rust
  awesome-tor-pr.md               same for ajvb/awesome-tor (or fork)
  awesome-decentralized-pr.md     same for gdamdam/awesome-decentralized-systems
  show-hn.md                      Hacker News "Show HN" post
  reddit-r-privacy.md             r/privacy launch post
  reddit-r-rust.md                r/rust launch post
  reddit-r-selfhosted.md          r/selfhosted launch post (relay angle)
  reddit-r-tor.md                 r/TOR launch post (very strict mod, follow rules)
  mastodon.md                     Fediverse toot drafts
  alternativeto.md                instructions to add balchat as alternative
  privacyguides-discussion.md     prep for the Privacy Guides forum
```

## Suggested order

1. **GitHub housekeeping done** — LICENSE, SECURITY, PRIVACY, README ✅
2. **IzzyOnDroid** — submit first; gives Android users a real install path
3. **Awesome lists** — submit PRs in parallel, low-risk
4. **AlternativeTo** — bulk inbound traffic, low effort
5. **Mastodon** — toot from `@xandru582@fosstodon.org`
6. **Reddit** — wait until at least IzzyOnDroid is live so users can try the app
7. **Show HN** — only when you're confident the app survives a few hundred concurrent installs (last 24 h before posting: smoke-test the relay, the website, the download links)
8. **Privacy Guides** — only after a few weeks of public use + at least one external review
